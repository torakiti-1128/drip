# プロジェクト概要

## 1. コンセプト
プロジェクト名はDrip (Daily Report Insights Pipeline) で、日報を成長へつなげるAI洞察パイプライの英文頭文字から命名。Markdown形式の日報を構造化された「要約」「洞察」「クイズ」「アクション」をに変換する自己成長基盤。

## 2. システムアーキテクチャ
リソースを最小限に抑えつつ、最大限の出力を得るための構成。

* **Frontend**: React (Vite) / Tailwind CSS / shadcn/ui
    * 直感的な日報プレビューとクイズ回答UI。
* **Backend**: Ruby on Rails 8 (API Mode)
    * Markdownのパース、Gemini APIとの連携、スケジュール実行。
* **Database**: PostgreSQL (Supabase Free Tier)
    * 日報履歴、生成されたクイズ、ユーザー設定の永続化。
* **AI Engine**: Gemini 2.0 Flash
    * 高速かつ広範なコンテキスト理解によるインサイト抽出。
* **Auth**: Supabase Auth (GitHub連携等)
    * 個人専用環境としてのセキュアなアクセス制限。

## 3.  基本機能

| ステージ             | 内容                                 | 実装のポイント                  |
| :--------------- | :--------------------------------- | :----------------------- |
| **1. Intake**    | Markdown日報のインポート（Web入力/特定ディレクトリ監視） | パーサーによるメタデータ（日付・タグ）の自動取得 |
| **2. Filter**    | AIによる情報の濾過（ノイズ除去と本質抽出）             | 事実と気付きの分離                |
| **3. Infuse**    | 学習定着のための「サマリー」「抽出クイズ」を生成           | 記憶の定着を促す「一問一答」と解説のセット    |
| **4. Discharge** | 次のアクションと余談の出力                      | 翌日のタスクリスト自動作成とモチベーション維持  |
| **5. Notify**    | 成果の通知（Webhook連携）                   | 指定した時間（朝など）に前日の振り返りを通知   |

## 4. 管理機能
* **Pipeline Editor**: AIへのプロンプトをGUI上でリアルタイムに調整。
* **Format Designer**: 日報入力テンプレートおよび出力フォーマットのカスタマイズ。
* **Schedule Manager**: 振り返りサマリーの生成頻度や通知タイミングの設定。
* **Plugin Service**: 使用するAIや通知先を簡単に変更可能。

## 5. 戦略的マイルストーン
1.  **Phase 1 (MVP)**: RailsからGeminiを叩き、Markdownを投げるとサマリーとクイズが返る最小構成。
2.  **Phase 2 (UI)**: Reactでの管理画面実装、プロンプトの動的変更機能。
3.  **Phase 3 (Automation)**: 定期実行機能と、外部通知（Slack等）の統合。

## 6. その他機能
- **余談機能**: AIによる「意外な視点からのコメント」をあえて組み込むことで、アプリを開く動機付けを強化する。
- **データポータビリティ**: 抽出した知見をMarkdown形式で再度エクスポート可能にし、個人のナレッジベース（ObsidianやNotion）へ還流させる導線を確保する。
